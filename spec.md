# NTU 經濟系研究所學門歸類工具 — 規格書

## 專案概述

給台大經濟系碩士生使用的純前端網頁工具，協助學生輸入修過的課程、自動對應所屬學門，並找出所有符合畢業門檻（2 個不同學門、每學門至少 2 科）的組合，最後可匯出結果。

---

## 技術架構

- **純前端**：單一 HTML 檔案（內嵌 CSS + JS），無後端、無資料庫
- **課程資料**：獨立 `courses.json` 資料檔，與主程式分離，方便每年更新
- **部署**：可直接用 GitHub Pages 免費托管
- **匯出**：產生純文字或 CSV 讓使用者下載或複製

## 功能規格

### 1. 課程輸入區

- 預設顯示 1 個輸入列，可動態新增／刪除列，無上限
- 每一列包含：
  - **課號欄**：文字輸入＋下拉搜尋，輸入課號（如 `ECON5117`）後：
    - 即時 fuzzy 搜尋，顯示候選清單
    - 選定後自動填入課名（中英文）
  - **課名欄**：文字輸入＋下拉搜尋，輸入中文或英文課名後：
    - 即時搜尋，顯示候選清單
    - 選定後自動填入課號
  - **所屬學門欄**（唯讀，自動帶出）：顯示該課所屬的所有學門，以 tag 形式呈現
  - **刪除按鈕**：移除該列
- 課號與課名任一欄輸入皆可觸發另一欄自動填入
- 若輸入內容找不到對應課程，顯示提示「查無此課程，請確認課號或課名」
- 防止同一課程重複輸入（若已存在則提示並阻擋）

### 2. 學門歸類分析

- 使用者填完所有課程後，點擊「**學門歸類**」按鈕
- 系統計算所有可達到畢業門檻的組合，規則：
  - 選出 2 個不同學門
  - 每個學門各選 2 門課（不少於）
  - 同一門課不可同時計入兩個學門
- 結果呈現：
  - 列出**所有合法組合**
  - 每個組合顯示：學門 A（課程 1、課程 2）＋ 學門 B（課程 1、課程 2）
  - 若同一課屬於多個學門，同一組合內該課只能歸入一個學門
  - 若無任何合法組合，顯示提示並說明差距（例如：「總體與貨幣 只有 1 門課，還差 1 門」）

### 3. 結果摘要區（分析後顯示）

顯示兩個區塊：

**A. 學門完整度一覽表**

| 學門 | 已修課程數 | 狀態 |
|------|-----------|------|
| 總體與貨幣 | 3 | ✅ 可用 |
| 計量經濟 | 1 | ⚠️ 差 1 門 |
| 人力資源 | 0 | — |

**B. 所有合法組合**

每個組合以卡片形式呈現，使用者可點選「選用此組合」標記偏好，供匯出使用。

### 4. 匯出功能

點擊「**匯出結果**」按鈕，提供兩種方式：
- **下載 CSV**：含所有已輸入課程（課號、中文課名、英文課名、所屬學門）＋選用的學門組合
- **複製文字**：將選用組合格式化為可貼入表格的純文字，例如：

```
學門一：總體與貨幣
  - ECON5177 動態隨機一般均衡模型
  - ECON7182 應用總體經濟學

學門二：計量經濟
  - ECON5176 計量經濟學一
  - ECON5164 計量經濟學二
```

## UI/UX 規格

### 語言與版面
- 語言：中文介面為主，課名同時顯示中英文
- 版面：單頁，上方輸入區、下方結果區
- RWD：支援手機與桌機
- 下拉搜尋元件：自製或使用輕量套件（如 Tom Select 或 Choices.js）
- 介面中**禁止使用任何 emoji**，狀態文字改用純文字標示，例如「可用」、「不足」

### 色系（參考台大經濟系官網）
| 用途 | 色碼 |
|------|------|
| 主色（導覽列、標題） | `#005073`（深藍綠） |
| 輔色（強調、按鈕hover） | `#8b3a2a`（紅棕） |
| 背景 | `#ffffff` |
| 次要背景（表格偶數列、區塊） | `#f4f6f7` |
| 文字主色 | `#1a1a1a` |
| 邊框、分隔線 | `#cccccc` |

### 設計風格要求（反模式清單）
以下為明確**禁止**的設計手法，避免產生 AI vibe coding 的視覺感：

- **禁止** `border-radius` 超過 3px（無大圓角卡片）
- **禁止**使用任何漸層色（`gradient`）
- **禁止**使用 `box-shadow` 製造浮起感
- **禁止**使用動畫或過渡效果（`transition`, `animation`）超過功能需要
- **禁止**使用大型 hero banner 或全寬背景圖
- **禁止** padding 過大造成大量留白的「現代感」版面

整體風格參考學術機構／政府機關網站：清晰、方正、格線感強、資訊密度適中。具體參照 `https://econ.ntu.edu.tw/` 的排版邏輯：水平導覽列、細邊框表格、無裝飾性元素。

## courses.json 生成說明

Claude Code 應自行從本規格書附帶的 PDF（`Graduate_field_coures.pdf`）解析所有課程資料並生成 `courses.json`。

生成完畢後，請 Claude Code 輸出以下驗收統計供人工核對：
- 總課程筆數
- 每個學門各有幾筆課程
- 有哪些課程屬於 3 個以上學門（列出課號）

若同一課號出現多個課名變體，以 PDF 原文為準，取最完整版本。

---

## 資料維護說明（給未來更新者）

只需編輯 `courses.json`：
- 新增課程：新增一筆 JSON 物件
- 修改學門對應：調整該課的 `fields` 陣列
- 刪除課程：移除對應物件

不需修改 HTML / JS 主程式。
